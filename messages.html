Cara Membuat Alias di Termux

Kalau anda pernah mengoperasikan command-prompt atau cmd di Windows, tentu anda familiar dengan perintah-perintah seperti dir, ren, del dan sebagainya.

Bisakah kita membuat perintah-perintah seperti itu di Termux?

Jawabnya bisa, yaitu menggunakan alias. Contoh, dir (directory) akan dialiaskan kepada ls (list), ren (rename) akan dialiaskan kepada mv (move) dan del (delete) dialiaskan kepada rm (remove). Bahkan kalau mau, kita juga bisa membuat perintah-perintah sendiri sesuka kita dengan alias.

Bagaimana caranya?

1. Buka Termux

2. Ketik:

nano /data/data/com.termux/files/usr/etc/bash.bashrc

3. Tambahkan baris berikut di akhir file:

#ini adalah alias buatanku sendiri

alias dir="ls"
alias ren="mv"
alias del="rm"

4. Simpan perubahan tersebut dengan menekan CTRL+S, lalu keluar dengan menekan CTRL+X.

5. Restart Termux (caranya ketik exit, lalu enter, setelah itu buka lagi seperti biasa)

Sekarang alias sudah bisa dicoba.

Cara Menghapus Alias

Untuk menghapusnya, cukup pakai unalias, contoh:

unalias dir
unalias ren
unalias del

Atau kalau mau menghapus semuanya:

unalias -a

Referensi:

https://www.tecmint.com/create-alias-in-linux/amp/

Cara membuat file yang bisa dieksekusi langsung di Termux dengan bahasa PHP

Biasanya, untuk menjalankan file PHP di Termux kita mengetik perintah php namafile.php (atau php /lokasi/penyimpanan/namafile.php).

Kali ini kita akan menjalankan file tersebut tanpa menggunakan perintah php, bahkan tanpa menuliskan path (lokasi)nya dan tanpa akhiran .php, misalnya index, salam dan sebagainya.

Sebagai contoh, kita akan membuat di dalam direktori home (~/)

Caranya:

1. Ketik cd lalu enter (supaya masuk ke direktori home). Kemudian bikin folder (direktori baru) bernama .bin (sengaja diberi titik sebelumnya supaya menjadi folder yang tersembunyi). Begini caranya:

$ mkdir .bin

2. Bikin file bernama .bash_profile yang isinya adalah PATH=\$HOME/.bin:\$PATH

$ echo "PATH=\$HOME/.bin:\$PATH" >> ~/.bash_profile

3. Pindahkan file PHP yang mau dieksekusi ke folder .bin tersebut, misalnya kita akan memindahkan file index.php dan mengubah namanya menjadi salam

$ mv ~/index.php ~/.bin/salam

4. Berikan akses eksekusi pada file tersebut:

$ chmod u+x ~/.bin/salam

5. Buka file salam. Pada baris pertama, yaitu di atas <?php sisipkan baris berikut: #!/data/data/com.termux/files/usr/bin/env php

$ nano ~/.bin/salam

Jadinya seperti ini:

#!/data/data/com.termux/files/usr/bin/env php

<?php

echo "Assalamu'alaikum".PHP_EOL; # ini contoh saja

Simpan perubahan tersebut dengan menekan CTRL+X lalu tekan huruf y (yes) lalu tekan enter.

6. Ketik perintah berikut:

$ source ~/.bash_profile

Artinya, jadikan file .bash_profile yang ada di home (~/) tersebut sebagai sumber (source) program yang akan dieksekusi.

Selesai.

Untuk memanggilnya, cukup ketik salam lalu enter, maka file yang isinya script PHP tersebut akan langsung dieksekusi. Info lebih lanjut tentang perintah source: ketik source --help

Contoh lain, buat file sapa yang isinya:

#!/usr/bin/env php
<?php
$name = isset($argv[1]) ? $argv[1] : 'php';

echo "Hai $name" . PHP_EOL;

Cara memanggilnya:

$ sapa Budi
Output: Hai Budi

Cara Mengendalikan Termux dari Komputer Lain

Termux adalah terminal mungil yang handal di Android. Dengannya kamu bisa menjalankan berbagai perintah dasar Linux. Kali ini kita akan belajar cara memasang Server SSH di Termux supaya bisa diakses dan dikendalikan dari komputer lain melalui jaringan.

Ok anggap saja tanda $ (Dollar) sebagai Termux dan tanda # (pagar) sebagai komputer lain.

Pertama, kamu perlu memasang paket-paket yang diperlukan. Pastikan semuanya sudah di-update ya.. caranya jalankan perintah berikut:

$ pkg upgrade

Kemudian, pasang paket OpenSSH dengan menjalankan perintah berikut:

$ pkg install openssh

Berikutnya, cara memulai dan mengentikan Server SSH. Untuk memulai Server SSH, jalankan perintah berikut:

$ sshd

Untuk menghentikannya, gunakan perintah berikut:

$ pkill sshd

Oke, selanjutnya adalah memasang password untuk otentifikasi.

Idealnya, kita harus memasang key untuk otentifikasi. Tapi untuk memudahkan latihan, kita lewati saja lengkah itu, tapi perlu diingat bahwa password itu lebih rendah tingkat keamanannya daripada key. Jadi, kalau kamu mau lebih aman, pakailah key.

Secara default, password sudah terpasang di Termux, tapi kamu bisa meninjau ulang pengaturan tersebut dengan menjalankan perintah berikut:

$ cat $PREFIX/etc/ssh/sshd_config

Nanti penampilannya kurang lebih seperti ini:

PrintMotd yes
PasswordAuthentication yes
Subsystem sftp /data/data/com.termux/files/usr/libexec/sftp-server

Untuk memasang password, jalankan perintah berikut:

$ passwd

New password: (ketik saja meskipun tidak muncul apa-apa di layar, lalu tekan enter)

Retype new password: (ulangi lagi dengan password yang sama dengan di atas)

Kalau berhasil akan muncul kalimat begini:

New password was successfully set.

Selanjutnya, kamu harus tahu alamat IP hp Android kamu, caranya jalankan perintah berikut:

$ ifconfig

Dari hasil di atas, kita dapati bahwa alamat IP Android kamu di jaringan adalah 192.168.100.92

Ok, sampai di sini kamu sudah siap memulai Server SSH dan siap terhubung dengan perangkat Android kamu.

Sekarang kita akan mulai mengendalikan Termux dari komputer lain.

Buka terminal Linux di komputer lain, lalu ketikkan perintah berikut:

# ssh apasaja@192.168.100.92 -p 8022

Kamu bisa pakai username apa saja sebab Termux tidak peduli dengan username. Dalam contoh ini, saya pakai username apasaja.

Setelah itu kamu akan diminta memasukkan password yang sudah kamu pasang di Android di atas. Setelah itu, tara.. kamu sudah berhasil masuk ke dalam Termux melalui komputer lain.
Sekian, semoga bermanfaat. Selamat mencoba.

Cara Membuat Form Login di Termux

Demi keamanan, kadang seseorang perlu mengunci Termux-nya dengan password supaya tidak bisa diakses oleh sembarang orang. Berikut ini cara membuatnya:

1. Bikin script login.sh dengan mengetikkan perintah berikut:

$ nano login.sh

2. Masukkan script berikut (ubah abc123 dengan password yang kamu inginkan):

#!/data/data/com.termux/files/usr/bin/env bash

clear;
echo
passwod=
user="user"

handle_ctrl_c() {
if [[ $EUID != 0 ]]; then
sudo pkill com.termux
else pkill com.termux
fi
}
trap "handle_ctrl_c" 2

while true; do
read -s -p "masukkan password: " passwod
if [[ $passwod != "abc123" ]]; then
echo "password salah!"; sleep 1; clear;
else
break
fi
done;
clear

3. Simpan (dengan cara menekan tombol CTRL + S), lalu keluar (dengan cara menekan tombol CTRL + X).

4. Berikan akses eksekusi pada file tersebut:

$ chmod +x login.sh

5. Edit file bash.bashrc dengan mengetikkan perintah berikut:

$ nano /data/data/com.termux/files/usr/etc/bash.bashrc

6. Masukkan baris berikut di baris paling akhir (paling bawah)

/lokasi/file/login.sh

Ubah /lokasi/file/ dengan lokasi sesungguhnya. Misal, kalau kamu menyimpannya di direktori home berarti menjadi ~/login.sh

7. Simpan file tersebut (CTRL+S) dan tutup (CTRL+X).

8. Oke selesai, sekarang restart Termux kamu dan sudah ada form untuk memasukkan password.

Cara Menyingkat Perintah chmod +x dengan Shell Script

Kita sering menggunakan perintah chmod +x untuk mengubah permission suatu file menjadi executable (bisa dieksekusi).

Bisakah perintah itu kita singkat saja menjadi cx misalnya?

Jawabnya, sangat bisa. Begini caranya:

1. Bikin file bernama cx

$ nano cx

2. Copy dan paste kode di bawah:

#!/usr/bin/bash

if [ $1 ]; then
chmod +x $1
else
echo "cx nama_file (chmod +x nama_file)"
fi

3. Simpan file tersebut (CTRL+S) lalu keluar (CTRL+X)

4. Jadikan file cx tersebut executable:

$ chmod +x cx

5. Pindahkan file cx tersebut ke folder PATH (untuk melihatnya, ketik echo $PATH). Contoh, PATH saya adalah ~/.bin, maka saya ketik:

$ mv cx ~/.bin

Selesai.

Untuk menggunakannya cukup ketik cx nama_file

Sedikit Tentang Customisasi Termux

Termux secara default memiliki warna latar belakang (background) yang hitam pekat dengan tulisan (foreground) berwarna putih cerah. Seperti berikut.

background=#000000 (hex color untuk hitam)
foreground=#ffffff (hex color untuk putih)

Warna tersebut memanglah umum yang mana notabene nya Termux merupakan terminal emulator.

Tapi bagi user yang bosan dengan warna default Termux, hal tersebut bisa diubah sesuai keinginan.

Termux memiliki direktori spesial yang terletak di folder $HOME. Direktori ini diawali dengan titik (dot), dengan nama '.termux' (dot termux).

Direktori ini secara otomatis terbuat saat pertama kali user menginstal Termux. Namun pada versi Termux yang lebih lawas, direktori spesial ini tidak secara otomatis terbuat. Tetapi masih bisa dibuat secara manual oleh user menggunakan command:

$ mkdir ~/.termux

Pada direktori inilah user bisa mengatur warna yang ada pada Termux. Pengaturan warna tersebut terletak pada file yang bernama 'colors.properties'.

Jika file colors.properties belum ada, user dapat membuatnya dengan command:

$ touch ~/.termux/colors.properties

Sebagai contoh, dibawah ini adalah konfigurasi warna favorit saya yang saya terapkan pada Termux.
Bisa dicoba diterapkan nih.

background=#141821
foreground=#abb2bf
cursor=#abb2bf

color0=#282c34
color1=#e06c75
color2=#98c379
color3=#e5c07b
color4=#61afef
color5=#c678dd
color6=#56b6c2
color7=#abb2bf

color14=#56b6c2
color15=#c8ccd4

Dengan menggunakan text editor, copy dan paste kan baris diatas ke dalam file 'colors.properties'.
Jika menggunakan nano:

$ nano ~/.termux/colors.properties

Lalu simpan file tersebut dengan menekan ctrl + s.
Kemudian reload Termux untuk menerapkan perubahan dengan menggunakan command:

$ termux-reload-settings

Seperti yang telah disebutkan sebelumnya, "background" mengatur warna latar belakang dan "foreground" mengatur warna tulisan.
"cursor" mengatur warna dari pointer nya.

Sedangkan "color0"-"color15" adalah pengaturan warna lainnya. Misalnya saja ketika user mengeksekusi perintah 'ls' untuk menampilkan list dari isi sebuah direktori, maka list tersebut dibedakan dengan warna sesuai tipe file masing-masing.

Direktori berwarna biru, file executable berwarna hijau, symlink berwarna cyan, dsb.

Mungkin itu dulu yang bisa saya bagikan.

Jika ada yang ingin ditanyakan silahkan berdiskusi di kolom diskusi/komentar.

Sekian, semoga bermanfaat. :)

Cara Mengetahui Alamat IP Kita dengan Cepat

Biasanya, kalau mau lihat IP kita kirim perintah ip addr atau ifconfig. Tapi masalahnya, data yang ditampilkan sangat banyak. Padahal yang kita perlukan cuma alamat IP saja. Bagaimana caranya supaya yang muncul cuma alamat IP kita saja?

Caranya, bikin alias seperti ini:

$ alias ipku="echo $(ifconfig | grep broadcast | awk '{print $2}')"

Untuk memanggilnya, cukup ketik:

$ ipku

Maka akan keluar alamat IP anda.

Cara Mencari Hadis pakai Termux

$ ch Shahih_Bukhari:1

Cara Transfer File dari Termux ke Linux

Langsung saja, untuk Termux kita beri tanda Dollar ($), untuk Linux kita beri tanda pagar (#)

1. Aktifkan dulu SSH Server di Linux:
# sudo service ssh start

2. Kirim file dari Termux. Formatnya: scp [nama_file] username@ip:/lokasi

Contoh:

$ scp catatan.txt aku@192.168.0.1:/home/aku

Kalau dari Termux ke Termux, gunakan port 8022:

$ scp -P 8022 catatan.txt a@192.168.0.1:/sdcard

Cara Backup dan Restore Termux

Pada contoh ini, kita akan backup direktori home dan usr. Hasilnya akan disimpan di /sdcard dan dikompres pakaiÂ gzip.

Pertama: Backup

1. Pastikan Termux sudah diberi izin mengakses storage:

$ termux-setup-storage

2. Pergi ke direktori utama Termux:

$ cd /data/data/com.termux/files

3. Mulai backup file:

$ tar -zcvf /sdcard/termux-backup.tar.gz home usr

Seharusnya backup akan berjalan lancar tanpa eror.

Kedua: Restore

Anggap kamu sudah berhasil membackup direktori home dan usr. Sekarang kamu akan merestorenya kembali. Perlu diingat bahwa dalam proses restore, file-file yang ada akan ditimpa dengan file-file yang baru. Jadi, harap hati-hati.

1. Pastikan Termux sudah diberi izin mengakses storage:

$ termux-setup-storage

2. Pergi ke direktori utama Termux:

$ cd /data/data/com.termux/files

3. Mulai ekstrak:

$ tar -zxf /sdcard/termux-backup.tar.gz --recursive-unlink --preserve-permissions

Setelah itu tutup Termux dan buka kembali.

Selesai.

# Part 1 - Instalasi dan Dasar Vim
--------------

Vim atau Neovim adalah code editor paling tepat untuk digunakan di Termux.

Sebab Vim memakai kontrol navigasi yang sangat simpel, mendukung banyak manipulasi string dan juga word-completion!

Vim juga sangat customizable untuk user mengatur keybinding ataupun mapping miliknya sendiri.
Makanya Vim juga sangat terkenal dan digunakan oleh banyak programmer profesional.

NOTE: untuk info Vim lanjut silahkan googling yak

langsung saja...

Instal Vim menggunakan perintah:
$ pkg install neovim

Untuk memulai menggunakan Vim ketikkan:
$ nvim
atau
$ nvim nama_file

Maka akan ditampilkan tampilan awal Vim dalam mode NORMAL.

Untuk mulai menulis di dalam Vim, silahkan masuk ke dalam mode INSERT dengan mengetikkan i.

Untuk rincian nya silahkan lihat manpage, :help command, atau silahkan googling :)

Membuat Program Sederhana Penghitung Luas

Cara menghitung luas adalah dengan mengalikan panjang dengan lebar suatu bidang. Rumusnya seperti ini:

Luas = Panjang x Lebar

Contoh, jika panjang 5 dan lebar 4 maka luasnya adalah:

5 x 4 = 20

Sekarang kita mau bikin program sederhana untuk melakukan tugas itu dengan menggunakan Termux.

Caranya mudah:

1. Pastikan terlebih dahulu di mana lokasi file sh lalu salin lokasi tersebut:

$ which sh

/data/data/com.termux/files/usr/bin/sh

2. Bikin file hitungluas.sh

$ cat > hitungluas.sh

3. Copas kode di bawah ini:

#!/data/data/com.termux/files/usr/bin/sh
echo "Panjang: "
read PANJANG
echo "Lebar: "
read LEBAR
LUAS=$(expr $PANJANG \* $LEBAR)
echo "Luas: $LUAS"

4. Tekan tombol CTRL+C

^C

5. Jalankan file hitungluas.sh

$ sh hitungluas.sh

6. Masukkan panjang dan lebar. Contoh kita akan masukkan 4 untuk panjang dan 5 untuk lebar. Jangan lupa tekan enter setelahnya.

Panjang:
4
Lebar:
5
Luas: 20

Oke, berhasil.

Kalau mau menghitung lagi, cukup panggil dengan perintah:

$ sh hitungluas.sh

Lalu masukkan datanya seperti di atas.

Selamat mencoba.

Cara Mudah Membuat Bot dengan Termux

Bikin bot gampang banget, gini caranya:

1. Daftarkan nama botmu di @BotFather (setelah menekan tombol START, kirim /newbot lalu ikuti instruksi yang diberikan). Kalau sudah, kamu akan diberi username dan token.

2. Buka Termux lalu bikin direktori, namanya terserah, misal: botku, lalu masuk ke dalamnya:

$ mkdir botku
$ cd botku

3. Ambil script atau source code dari github:

$ git clone https://github.com/dannsbass/botphp

Tunggu sampai selesai.

4. Cek apakah sudah ada direktori botphp:

$ ls

5. Kalau ada direktori bernama botphp masuk ke dalamnya:

$ cd botphp

Di dalamnya ada 3 file, yang kita pakai cuma 2 file saja, yaitu data.txt dan bot.php

6. Buka data.txt:

1. Bikin file bernama hitungfile.sh

2. Copas kode di bawah ini:

#!/usr/bin/bash
declare dirpath="$PWD"
declare -a filepaths=("$dirpath"/*)

# Print filenames +/- directory

for f in "${filepaths[@]}" ; {
# printf %s\\n "${f##*/}" # Filename
printf %s\\n "$f" # Full path
}

# File / folder counts

declare -a files=() dirs=()
for f in "${filepaths[@]}" ; {
[ -f "$f" ] && files+=("$f") || {
[ -d "$f" ] && dirs+=("$f") ; }
}

printf "Number of files: %d\n" "${#files[@]}"
printf "Number of folders: %d\n" "${#dirs[@]}"

Cara Mendeploy Aplikasi di Heroku Melalui Termux

Deploy artinya mengangkat dan mengangkut. Dalam dunia pemrograman, deploy bermaksud mengunggah file-file yang sudah siap dipakai ke server. Perbedaan deploy dengan upload, kalau deploy biasanya disertai pencatatan terhadap perubahan versi (version control) sehingga memudahkan kita untuk kembali ke versi sebelumnya kalau diperlukan. Sedangkan upload akan mengganti file lama dengan file baru tanpa ada kontrol terhadap perubahan versi.

Oke langsung saja, cara deploy aplikasi di Heroku sangat mudah.

$ git init

Ketiga, lakukan pengendalian aplikasi di heroku dengan mengetik perintah: heroku git:remote -a nama-aplikasimu

$ heroku git:remote -a dannsbass

$ git add .
